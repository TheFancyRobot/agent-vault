## Human Notes

- Vocabulary matters: these are "interface stubs" / "source interface skeletons", not "pseudo-code stubs". A stub must never invent or approximate behavior — it keeps real declarations and drops bodies.
- Stubs generated by a weaker fallback extractor must be clearly marked as incomplete/fallback so consumers can decide to fetch fuller content.
